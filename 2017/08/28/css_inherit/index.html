层叠和继承

Share

层叠和继承，是 css 两大核心概念，当样式表越写越复杂，你可能会开始考虑为啥页面中的元素会显示成那样？有时样式又会冲突，也就是一个元素被多个规则声明了相同的 css 属性。幸好 css 会使用一种机制处理掉冲突，这种机制就叫做“层叠”，用于管控样式间相互作用的方式，出现冲突时判断哪个样式的优先级高，就执行哪一种。

样式层叠的方式

层叠指一系列的规则，用来确定究竟把哪种样式属性应用到元素身上。浏览器通过层叠规则，判断如何处理一个元素上的多个样式，以及何如处理有冲突的属性。这两种情况都会导致样式冲突：

  1. 继承
  2. 一个元素拥有多个样式

继承

接着我们再来看看“继承”。小孩会从父母处继承一些身体特征，比如发色、瞳孔色、身高，或者…财富😂等等。有时还会隔代继承。和人类相似，html 标记也会从祖辈那里继承一些 css 属性。

什么是继承

继承是指应用在某个标记上的 css 属性传给了内部嵌套的标记。比如说 <div>、<p> 等等都始终嵌套在 <body> 里，所以应用在 <body> 标记上的属性会被 <div> 和 <p> 继承。比如给 <body> 设置了 color: red 那么 <body> 里的文字标记就会渲染为红色。

css 属性也会隔代继承。如果 <p> 标记还有 <span> 或者 <em> 或者其他，也会继承应用到 <body> 标记上的样式。

继承可以为开发节省很多时间，如果没有继承，我们会一次一次声明相同的样式。继承不仅适用于标签样式，如果在某个标记上应用类、或者是后代选择器等样式，标记里子级会从类样式中继承属性。

局限性

人无完人，继承也是一样，很多 css 属性不会传给后代标记，而这并不一定就是坏事，如果继承了某些属性，会对网页的外观产生负面的影响。像 border 、margin、background 等就不会继承。

哪些属性会被继承

  • color
  • font

    • font-size
    • font-style
    • font-variant
    • font-weight
  • letter-spacing
  • word-spacing
  • line-height
  • text-align
  • text-decorition
  • text-indent
  • text-shadow
  • text-transform
