CSS里尺寸单位的小事

Share

在书写样式表时都会与单位关联，例如 .container{width: 100px;} 或者是 .left{width: 50%;} 似乎常用到的单位也就那么两个，而实际上 css 中的可用单位的数量还包含了其他。除去绝对单位和常用的 px 、 % ，这篇博还将介绍几种你也许从没用过或见过的计量单位。可以通过 caniuse 这个网站查看其兼容性。

em

首先来看看 em 这个单位，浏览器默认渲染的文字大小是 16px ，当然了，我说的是在没有经过人为更改过设置的情况下。em 是一个相对大小，值可以设置为 1em 1.5em 0.5em 等，还可以指定到小数点后3位。而其中“相对”的意思是指：

首先需要明白的一点在于，rem 中的这个 “ r “ 指的是 root 也就是根元素 html ，酱紫一来就解释得通了，只需要更改 html 的 font-size 值，其他元素的值就基于它改变了。

vw 和 vh

有时你想要使用显示窗口的宽度或者高度，而不使用父级的宽度，该如何设置呢？这时 vw 和 vh 就闪亮登场了。

1vw 等于 viewport 宽度的 1%， 1vh 等于 viewport 高度的 1% 。也就是 window.innerWidth 与 window.innerHeight 的大小。

比如说一款浏览器的宽度是 750px，那么 1vw 的值就是 7.5px，同理，如果一款浏览器高度是 900px ，那么 1vh 就能得到 9px

那么我们就可以利用这个单位实现占满高度的图片轮播

vmin 和 vmax

vh 和 vw 总是与视口的宽高有关的，与之不同的 vmin 和 vmax 是与宽高的最大值或最小值有关的，取决于哪一个更大或更小。

比如说当前浏览器 width=1000px，height=500px，500 < 1000，那么 1vmin 就是 5px，1vmax 就是 10px。然而，如果 width=500px ，height=1000px，那么 1vmin 就是 10px，1vmax 就是 5px
